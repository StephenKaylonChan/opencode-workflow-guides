# 自动化配置指南

> 用 OpenCode 的 `opencode.json`、permissions、formatters、plugins 代替 Claude Code Hooks

**版本**: v1.3
**适用**: OpenCode（2026 年）

---

## 1. 自动化设计总览

在 OpenCode 中，自动化主要拆成四部分：

| 目的 | OpenCode 机制 |
|------|---------------|
| 规则注入 | `AGENTS.md` + `instructions` |
| 安全边界 | `permission` |
| 写后整理 | `formatter` |
| 生命周期处理 | `plugins` |

因此迁移时不要机械寻找“Hook 等价物”，而要改成：
- 能用 permission 解决的，不写插件
- 能用 formatter 解决的，不写插件
- 只有真正的生命周期逻辑才写 plugin

---

## 2. `opencode.json` 的职责

项目级配置建议放在根目录：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5",
  "instructions": [
    "docs/architecture/README.md",
    "docs/roadmap/README.md"
  ],
  "permission": {
    "bash": "ask",
    "edit": "ask",
    "webfetch": "ask"
  }
}
```

常用字段：
- `instructions`
- `permission`
- `formatter`
- `agent`
- `mcp`
- `tools`
- `snapshot`
- `share`

### 实战上建议再关注的字段

- `small_model`
- `compaction`
- `watcher`
- `plugin`

## 3. Permissions 设计

### 推荐原则

- 默认从 `* = ask` 开始更稳
- `bash`、`edit`、`skill` 都应该显式配置
- 对高频安全命令做 allow，对危险命令做 deny
- 危险行为可以再配合插件做二次拦截

### 示例

```json
{
  "permission": {
    "*": "ask",
    "read": {
      "*": "allow",
      "*.env": "deny",
      "*.env.*": "deny",
      "*.env.example": "allow"
    },
    "bash": {
      "*": "ask",
      "git status*": "allow",
      "git diff*": "allow",
      "git log*": "allow",
      "grep *": "allow",
      "rg *": "allow",
      "git push *": "deny",
      "git reset --hard*": "deny",
      "rm *": "deny"
    },
    "skill": {
      "*": "ask",
      "docs-refresh": "allow",
      "audit-checklist": "allow"
    }
  }
}
```

注意：
- OpenCode 默认已拒绝读取 `.env`，但在项目配置里显式写出来更清楚
- 权限规则支持模式匹配，且**最后一个匹配项生效**
- `external_directory` 应单独限制，避免模型无意跨项目读写

### 按 agent 覆盖

```json
{
  "agent": {
    "plan": {
      "permission": {
        "edit": "deny"
      }
    },
    "build": {
      "permission": {
        "edit": "ask"
      }
    }
  }
}
```

这相当于把“Plan Mode 不写代码”变成配置层约束。

### `external_directory` 建议

如果项目常访问工作区外目录，SHOULD 显式配置：

```json
{
  "permission": {
    "external_directory": {
      "~/projects/shared/**": "allow"
    },
    "edit": {
      "~/projects/shared/**": "deny"
    }
  }
}
```

## 4. Formatters 设计

OpenCode 支持配置 formatter，在写文件后自动整理代码。

### 示例

```json
{
  "formatter": {
    "prettier": {
      "command": ["npx", "prettier", "--write", "$FILE"],
      "extensions": [".ts", ".tsx", ".js", ".jsx", ".json", ".md"]
    },
    "black": {
      "command": ["python", "-m", "black", "$FILE"],
      "extensions": [".py"]
    }
  }
}
```

建议：
- formatter 只做“确定性整理”
- 测试、lint、通知、日志不要塞进 formatter

## 5. Plugins 设计

Plugins 才是 OpenCode 里最接近 Hooks 的能力。

可订阅的重要事件包括：
- `tool.execute.before`
- `tool.execute.after`
- `session.created`
- `session.compacted`
- `session.idle`
- `permission.asked`
- `todo.updated`
- `tui.prompt.append`
- `shell.env`

### 适合写 plugin 的场景

- 拦截对 `.env` 的读取
- 会话结束通知
- compaction 后记录日志
- shell 环境注入
- 自定义的安全检查
- 自定义 compaction 上下文

### 不适合写 plugin 的场景

- 单纯格式化
- 常规命令模板
- 单个任务步骤说明

## 6. 常用插件模板

### `.env` 保护

```javascript
export const EnvProtection = async () => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "read" && output.args.filePath.includes(".env")) {
        throw new Error("Do not read .env files")
      }
    }
  }
}
```

### 会话结束通知

```javascript
export const NotificationPlugin = async ({ $ }) => {
  return {
    event: async ({ event }) => {
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Session completed" with title "OpenCode"'`
      }
    }
  }
}
```

### 环境变量注入

```javascript
export const InjectEnv = async () => {
  return {
    "shell.env": async (input, output) => {
      output.env.PROJECT_ROOT = input.cwd
    }
  }
}
```

### compaction 上下文补强

```javascript
export const CompactionContext = async () => {
  return {
    "experimental.session.compacting": async (input, output) => {
      output.context.push(`
## Persistent State
- Current task status
- Important decisions
- Files being actively edited
`)
    }
  }
}
```

## 7. MCP 设计

OpenCode 原生支持本地和远程 MCP。

推荐原则：
- 只启用真正高频的 MCP
- 默认全局禁用重型 MCP，通过特定 agent 再开启
- 在 `AGENTS.md` 中明确何时使用哪个 MCP

### 示例

```json
{
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com",
      "enabled": true
    }
  },
  "tools": {
    "context7": false
  },
  "agent": {
    "research": {
      "tools": {
        "context7": true
      }
    }
  }
}
```

## 9. 插件依赖与本地包

如果本地 plugin 或 custom tool 需要 npm 包，OpenCode 支持在 `.opencode/package.json` 中声明依赖，并在启动时安装。

推荐做法：
- 只给 plugin/tool 需要的包建最小依赖
- 不要把整个项目依赖管理挪进 `.opencode/`
- 复杂 plugin 优先拆成独立 npm 包，再通过 `plugin` 字段加载

---

## 10. 迁移时的判断标准

把一个 Claude Hook 迁到 OpenCode 时，先判断它属于哪类：

| 旧 Hook 用途 | OpenCode 写法 |
|-------------|---------------|
| 提交前拦截 | `permission` 或 plugin |
| 写后格式化 | `formatter` |
| 会话开始提示 | plugin 或 command |
| 结束通知 | plugin |
| 自动注入上下文 | `instructions`、`AGENTS.md`、command |
| 保存进度 | command + spec/roadmap/session-notes |
